## Root Cause

### Root Cause 1: Synthetic QA DOM False Positive

`projects/002-claude-managed-agents/composition-remotion/scripts/prepare.mjs` generates an `index.html` with proxy markers such as:

- `info-row`
- `flow-token`
- `scan-fill`
- `metric-tick`

`scripts/review_video.mjs` and `scripts/qa_project.mjs` can count those markers and conclude that a scene has enough visual density, even when the actual Remotion render is visually sparse.

This creates a false positive:

```text
QA proxy DOM looks rich
-> review/QA passes
-> actual MP4 still looks empty
```

This must be blocked globally.

### Root Cause 2: Generic SceneFrame Became a Template Factory

`composition-remotion/src/scenes/SceneFrame.jsx` dispatches several scenes into generic components:

- `Synthesis`
- `BeforeAfter`
- `RubricLoop`
- `FanOutFanIn`
- `FlowPipeline`
- `SourceZoom`

These are useful primitives, but they became whole-scene templates. The result is that many scenes differ only by title and item list.

For a YouTube explainer, each important scene needs its own visual thesis and composition logic.

### Root Cause 3: Asset Status Means "Planned", Not "Useful On Screen"

`asset-plan.md` and `work-orders/*.md` currently mark routes as `qa_passed` even when:

- a capture is only used as vague background,
- the capture crop/highlight does not match the narration,
- the asset does not appear in the selected renderer,
- the screen can be understood without the asset, which means the asset route was not actually necessary.

Asset status must not pass just because a file exists or a route was written down.

### Root Cause 4: Motion Primitive Does Not Prove Motion Purpose

The current motion primitives can show cards, progress bars, rails, and loops. They do not always prove that the narration verb is visible.

Example failure:

```text
Narration says: "four features operate as one system"
Screen shows: four separate cards
```

That is not a design miss only. It is a contract miss.

## Generic Prevention System

These rules must apply to every future topic, not only project 002.

### Prevention Rule 1: Visual Thesis Before Render

Every scene needs a visual thesis before rendering.

No thesis means no render.

### Prevention Rule 2: Asset Route Must Be Visible In The Selected Renderer

`asset-plan.md` and `work-orders/*.md` are not enough.

Final QA must prove that the selected renderer uses the route in actual frames.

Examples:

| Route | Required Frame Evidence |
|---|---|
| `capture` | screenshot/crop/highlight visible and narratively relevant |
| `imagegen: primary` | generated image is the main visual carrier |
| `imagegen: support` | generated image supports mood or texture without replacing facts |
| `video-use` | processed clip appears in frame and timing matches packet |
| `hyperframes` / `remotion` | diagram/motion makes the narration verb visible |

### Prevention Rule 3: Machine Review Cannot Auto-PASS Taste

Machine review may block. It may not grant creative PASS.

Allowed machine verdict:

- missing artifact
- unresolved route
- caption too fast
- static gap
- repeated layout signature
- missing evidence frame
- likely empty panel
- floating line

Not allowed machine verdict:

- "looks good"
- "YouTube rhythm PASS"
- "motion purpose PASS"
- "director PASS"

Director PASS requires frame evidence inspected by `hype-video-reviewer`.

### Prevention Rule 4: No Proxy-DOM Quality Evidence

HTML markers are implementation hints only.

They cannot prove:

- visual density,
- asset presence,
- scene clarity,
- motion quality,
- caption safety.

Those must come from rendered MP4 frames.

### Prevention Rule 5: Scene Recipes Are Not Templates

Recipes define visual logic. They are not card layouts.

Bad:

```text
Use the same 4-card grid for every explain scene.
```

Good:

```text
Use fan-out/fan-in when the narration says work is split and merged.
Use document zoom when the narration points to a source or proof.
Use before/after when the narration reframes old way vs new way.
```

### Prevention Rule 6: Motion Must Show A Verb

Every motion must map to a narration verb:

| Verb | Motion Proof |
|---|---|
| combine | separate modules connect into one system |
| split | packets fan out from one source |
| merge | multiple results converge |
| judge | an output is evaluated against criteria |
| revise | failed output loops back and changes |
| trigger | event crosses a condition gate |
| remember | fragments sort into persistent memory |
| compare | before/after states change in place |

If motion cannot name its verb, remove or redesign it.
