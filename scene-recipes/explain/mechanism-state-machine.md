# explain/mechanism-state-machine

## Use When

- Explaining conditions, triggers, states, loops, or system behavior.

## Primary Routes

- `hyperframes`
- `remotion`

## Secondary Routes

- `capture` if a product or doc page proves the mechanism.

## Required Primitives

- `packetFlow`
- `anchoredPath`
- `liquidLens`

## Motion Beats

1. Input packet enters.
2. Condition gate checks it.
3. State changes visually.
4. Action/output appears.
5. Optional loop returns to the condition.

## Renderer Notes

- Hyperframes: keep paths short and anchored to concrete nodes.
- Remotion: use `Sequence` or local frame windows for state transitions.

## QA Fail If

- The state change is only text replacement.
- The trigger/condition/action relationship is not visible.
- Connector paths are decorative.
