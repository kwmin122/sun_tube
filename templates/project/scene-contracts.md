# Scene Contracts

Owner: `hype-scene-planner` with `hype-creative-director` and `hype-motion-designer`

Purpose: define the exact per-scene implementation contract so motion designers cannot fill unclear scenes with generic cards, fades, dots, lines, or glass panels.

Created after `timed-scene-packets.md`.

Validate with:

```bash
npm run factory:validate-scene-contract -- <project-path>
```

## Contract Rules

- A scene with missing required fields must return to planning.
- Only use visual elements listed in `Allowed Visual Elements`.
- Do not use elements listed in `Forbidden Fillers`.
- Do not use generic fallback components unless explicitly listed as allowed.
- Motion must prove the narration verb through the `Motion Beats`.
- Captions remain bottom safe-zone text-only from ElevenLabs SRT with no progress bar.
